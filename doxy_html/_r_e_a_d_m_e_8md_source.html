5 # ArnLib - Active Registry Network.
6 
7 This Qt based library makes it easy to distribute changing data objects. It also gives a
8 central place to find all your systems' current data. By using the ArnBrowser, all data
9 objects are real time presented in a tree view.
10 
11 ### Comparison to similar concepts
12 
13 * **Data mart:** Statistical data gathered from different systems. This makes it possible
14 to run cross system analysis.
15 
16 * **Windows Active Directory (R):** Centralized configuration data. All in one place easily shared.
17 
18 * **ArnLib:** Hot changing data from different systems. Enables easy cross system data
19 exchange, debugging, etc.

27 ArnLib could be beneficial in a lot of projects.
28 It should be well suited to the following conditions:
29 
30 * _A lot of configurations and changing values._ <Br>
31 ArnLib helps giving out-of-the-box diagnostics and ability to change values not yet
32 available in the custom application user interface.
33 
34 * _Hardware with a lot of sensors and controls._ <Br>
35 Arnlib helps giving a common interface and diagnostic.
36 
37 * _Distributed systems._ <Br>
38 ArnLib helps giving an out-of-the-box data sharing system that replicates Arn objects.
39 
40 * _Networked services by RPC (remote procedure call)._ <Br>
41 Will be quite the same as setting up signals and slots for local calls. You can find an
42 easy example in the ArnLib package, showing a simple chat Client and Server.
43 
44 * _ZeroConfig detection of present services._ <Br>
45 Helps advertise and browse a service (ftp, http, arn, ...) on a local network.
46 This is similar to UPNP discovery of units.

50 ## Main features
51 
52 * Based on Qt (4, 5 and 6), multiple platform and OS support.
53 * Qt based Arn browser available. Allows you to access all data objects in a tree view (see ArnBrowser).
54 * Web based Arn browser available, allowing you to use a standard web browser (see WebArnBrowser).
55 
56 #### Arn Data Objects
57 
58 * Hierarchical storage of hot changing data objects.
59 
60 * _Arn Data objects_ can be: integers, floats, strings, byte arrays and variants
61 (most Qt data types, e.g. QImage).
62 
63 * Data objects can typically be: measures, settings, data streams, documents, scripts (js), etc.
64 
65 * _Arn Data objects_ are thread-safe.
66 
67 * Native support for data validation and double direction pipes (streams).
68 
69 * Enums, Flags and SubEnunms in code available as text for output and parse.
70 
71 * Metrics of Arn available in Arn tree.
72 
73 #### Sharing
74 
75 * Data objects can be shared in a single program, among threads or between programs, at
76 different computers. This division of program modules can be changed and is transparent
77 to usage of ArnLib.
78 
79 * Support for temporary session data objects.
80 Optional auto-delete of objects when tcp/ip closes and unique uuid names.
81 
82 * Dependency system with custom offered services and getting signals when all needed services
83 are available.
84 
85 * Monitoring of newly created data objects and any mode change.
86 
87 * Login system, to give access protection and different privileges.
88 
89 * Remote access to Arn sessions, to view and control currently connected clients.
90 
91 #### Persistent storage
92 
93 * Optional persistent storage of object in SQLight or in a file.
94 
95 * Support for version control (VCS) of objects stored in files.
96 This can be git.
97 
98 #### Java Script
99 
100 * Native support in JavaScript for: _Arn Data Objects_, Dependency system and
101 Monitoring of changed objects.
102 
103 * Java Script jobstack with preemptive and cooperative scripts running at different priorities.
104 
105 #### Data streams and _Remote Procedure Call_
106 
107 * All data streams (pipes) can easily be monitored and manual test data can be inserted
108 (see ArnBrowser).
109 
110 * Service Api, for calling routines anywhere in connected Arn.
111 _Remote Procedure Call_ (RPC) simple to use as "remote signal slots".
112 
113 * Service Api has an automatically generated help for giving syntax when doing debug manual
114 typed calls to a RPC service.
115 
116 #### ZeroConfig and Discover
117 
118 * Any service (ftp, http, arn, etc) can be advertised, browsed and resolved for its host
119 address and port number.
120 
121 * High level, fully automatic support specialised for _arn_ service, can e.g. remotely
122  change the advertised _service name_.
123 
124 * Optional internal DNS_SD/mDNS routines for no dependency to any extra library.
125 
126 #### Qml
127 
128 * Support in Qml for: _Arn Data Objects_, monitoring of changed objects and Service Api (RPC).
129 
130 * Added support in Qml for url like "arn:///test.qml".
131 
132 * Possibility to create a remote generic Qml running environment, comparable to a web browser
133  running an arbitrary web application. This is done by ArnBrowser.
