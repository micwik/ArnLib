1 ArnLib Changelog / Todo
2 =======================
3 
4 Major
5 -----
6 * Script support for Sapi.
7 * ArnObject Link to other ArnObject (like in a filesystem).
8 * General access system with privileges at ArnObject level.
9 * Add more examples.
10 * Add Function tests.
11 * Add more Unit tests.
12 * API to Sync ArnObjects with other protocols (e.g. JSON-based).
13 * API to Sync ArnObjects over other media (e.g. CAN).
14 * Javascript based ArnLib for Web-applications over WebSocket.
15 
16 Minor
17 -----
18 * Optimize data transfer with minimal copying.
19 * Converter classes for ArnPipes to other streams (e.g UART, TCP etc).
20 * Addition to login a system to "pair" ArnServer and ArnClient.
21 
22 Done in 4.0
23 -----------
24 * Adopted to Qt6
25  Now with core5compat and some other deprecated parts.
26 * Added ArnCompat
27  This helps using same code for any Qt-version, relating to Arn based code.
28 * Added support for QJSEngine.
29  Previous support for QScript still works when using Qt version before Qt6.
30  Also some compatibility is handled by ArnScript.
31 * Added ArnScriptJobs general watchdog.
32  This is for QJSEngine but also backported to QScript.
33 * Added demo for ArnScriptJobs in examples.
34 * Added scriptauto to use in pro-file
35  This select QJSEngine or QScript depending on Qt-version. Prefered is QJSEngine.
36 * Added hostIp-list to ArnDiscover advertise.
37  This can be seen as new properties in ArnBrowser during "Arn Discover".
38 * Added ArnDiscover logic to also use "HostIp" property to select addr.
39  As MDns only can have 1 IP for a hostname, this helps chosing an connectable IP.
40 * Added SubEnum in MQFlags.
41  One or more Enums can be included in Flags as SubEnum.
42  This is also fully supported in text output, parse and XString-representation.
43 * EnumTxt is now a general usable class.
44  It can be used in the application for dynamic handling of Enums, Flags and SubEnums.
45 * Added XStringMap to QML.
46 * Added support for Enum (from Arn set) in QML.
47 * XStringMap has now options for minimizing length of XString.
48  This is backwards compatible and new formats are detected automaticly.
49  Options include: null-substitution (\0), repeted characters and a framing format.
50 * ArnSync is using new XStringMap optimization.
51  This is backward compatible. SyncVersion is changed to 4.0.
52 * XStringMap has new option AnyKey
53  When used, any key or value can be used, also binary blobs.
54 * Added atomic operations in ArnItem for: "bitSet" and "+=".
55  Single items are local and bidir works remotely by enabling a included provider.
56 * Added ArnMath
57  General math with templates and also some cpu-optimized
58  Includes: modulo, circleVal, isPower2, log2, minLim, maxLim, rangeLim
59 * Added ArnBasicItem isAssigning().
60  This can be used to cancel echo.
61 * Changed ArnDepend to be more robust and independent of ArnClient connection state.
62 
63 Done in 3.1
64 -----------
65 * Added ArnAdaptItem. Can be used in threads without eventloop or even non Qt threads.
66 * Added ArnClient syncMode for different client sync methods.
67 * Now all Bidir Objects has no echo, this was true only for pipes before.
68  The official value comes always from one provider. The requested value can be from many.
69 * Single objects has echo with better logic to avoid bad echoes that restores old values.
70 * Persistent values to client has more robust logic, especially for Master objects.
71 * Added ArnItem::setUncrossed(), will make it easier to build Arn Bridges etc.

73 Done in 3.0
74 -----------
75 * Delete ArnObject, but only local (remove any sync of it).
76 * ArnClient disconnect and close.
77 * Optimized memory consumption with pointers to different data in ArnLink.
78 * Minimized signal/slot:s in ArnLink by change to ArnEvent.
79 * Distributed deletion of folders.
80 * Distributed create of folder.
81 * ArnMonitor detects destructions of _Arn Objects_.
82 * Added setDelay in ArnItemQml, rework changed() and using timer events.
83 * Access system for Server/Client login with session level privilege.
84 * Allow read access to "freePaths" without login. Used to view for example licenses.
85 * Option for free nets, e.g. "localnet", that don't need login for full access.
86 * A flush mechanism for ArnPersist to force saving.
87 * Pimpl: Converted to d-pointer for making binary compatible library in the future.
88 * Started unit tests
89 * Optimized HandleData class with Null-state that can be this == 0.
90 * Made ArnObject (ArnLink) none QObject to save memory and independent on main-thread-create.
91  New methods and data for parent() etc.
92 * Changed to ArnLink::toInt(bool* isOk = 0).
93  To make ignoreSameValue work as expected for "" -> int=0 and similar. Same for all toXXX().
94 * Changed to ArnItem::toInt(bool* isOk = 0).
95  To give application the possibility to detect data type conversion errors.
96 * ArnBasicItem with no QObject, only inherited to give ArnEvent (QEvent). Small footprint!
97 * ArnItemNet (Arn syncing item) inherited from ArnBasicItem for small footprint.
98 * ArnMonitor no dependendency to ArnItemNet, that can be in other thread.
99 * ArnItem none native data-types: uint, int64 & uint64.
100 * Put ArnServer client sessions in "/Local/..." to be viewed and controlled (e.g kill).
101  Added ArnServerRemote class. Also chat between server (pipe in Arn) and client is supported.
102 * Browsing and controlling connected clients.
103 * Arn Registry metrics available in "/local/..."
104 * Added auto "humanize" logic to MQFlags text.
105  This will convert e.g. enum value WriteDelay200Ms to "Write delay 200 ms".
106 * XStringMap improved, e.g. addNumber().
