6 Introduction {#ins_intro}
7 ============
8 
9 This software uses qmake to build all its components.
10 qmake is part of a Qt distribution.
11 
12 qmake reads project files, that contain the options and rules how to
13 build a certain project. A project file ends with the suffix "*.pro".
14 Files that end with the suffix "*.pri" are included by the project
15 files and contain definitions, that are common for several project files.
16 
17 The first step is to edit the *.pri / *.pro files to adjust
18 them to your needs. Take care to select your deployment directories.

44 Building ArnLib {#ins_build}
45 ===============
46 
47 The software can be built both by command line and IDE (Qt Creator).
48 When using IDE, don't forget the "make install" step.
49 
50 
51 A) Unix {#ins_buildUnix}
52 -------
53 
54 > qmake <Br>
55 > make <Br>
56 > make install
57 
58 The easiest way of installing this library, is to let it be placed in a standard location
59 for librarys and includes, e.g. /usr/lib and /usr/include/ArnInc.
60 When using a shared library it's path has to be known to
61 the run-time linker of your operating system. On Linux systems read
62 "man ldconfig" (or google for it). Another option is to use
63 the LD_LIBRARY_PATH (on some systems LIBPATH is used instead, on MacOSX
64 it is called DYLD_LIBRARY_PATH) environment variable.
65 
66 If you only want to check the library examples without installing something,
67 you can set the LD_LIBRARY_PATH to the lib directory
68 of your local build.
69 it's also possible to compile the sources together by ArnLibCompile (see Using ArnLib below).
70 
71 The examples is built this way:
72 > cd examples/ArnDemoChat <Br>
73 > qmake <Br>
74 > make

150 Using ArnLib {#ins_usage}
151 ============
152 In the *.pro file of the application the below lines can be used.
153 
154 This will give a starting point for the configuration. It works well when using the same
155 base directory for ArnLib as the application, e.g. basedir/ArnLib and basedir/myApp. In
156 Unix-alike systems it's also needed to install the library files in a path known by the system,
157 see a) Unix.
158 
159 It's possible to include the ArnLib source in the application compiling by adding
160 ArnLibCompile to CONFIG. The included part of the source can be selected by addings to ARN,
161 e.g. ARN += server.
162 
163 WARNING! Using source inclusion (static linking) excludes the right to use LGPL for ArnLib.
164 Options are then to use GPL for the whole application or have a written agreement with
165 Michael Wiklund for other terms using the ArnLib.
166 
167 
168 Internal mDNS (ZeroConfig) is selected by adding mDnsIntern to CONFIG.
169 
170  CONFIG += ArnLibCompile
171  CONFIG += mDnsIntern
172 
173  greaterThan(QT_MAJOR_VERSION, 4) {
174  ARNLIB = Arn5
175  } else {
176  ARNLIB = Arn4
177  }
178 
179  ArnLibCompile {
180  #ARN += client
181  ARN += server
182  ARN += discover
183  include(../ArnLib/src/ArnLib.pri)
184  INCLUDEPATH += $$PWD/../ArnLib/src
185  } else {
186  win32: INCLUDEPATH += $$PWD/../ArnLib/src
187  win32:CONFIG(release, debug|release): LIBS += -L$$OUT_PWD/../ArnLib/release/ -l$${ARNLIB}
188  else:win32:CONFIG(debug, debug|release): LIBS += -L$$OUT_PWD/../ArnLib/debug/ -l$${ARNLIB}
189  else:unix: LIBS += -L$$OUT_PWD/../ArnLib/ -l$${ARNLIB}
190  }
191 
192  !mDnsIntern {
193  win32:CONFIG(release, debug|release): LIBS += -ldns_sd
194  else:win32:CONFIG(debug, debug|release): LIBS += -ldns_sd
195  else:unix: LIBS += -ldns_sd
196  }
197 
198 If you don't use qmake you have to add the include path to find the ArnLib
199 headers to your compiler flags and the ArnLib library to your linker list.
