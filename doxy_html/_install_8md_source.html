6 Introduction {#ins_intro}
7 ============
8 
9 This software uses qmake to build all its components.
10 qmake is part of a Qt distribution.
11 
12 qmake reads project files, that contain the options and rules how to
13 build a certain project. A project file ends with the suffix "*.pro".
14 Files that end with the suffix "*.pri" are included by the project
15 files and contain definitions, that are common for several project files.
16 
17 To use a more automated config of ArnLib and related applications,
18 the Qt Feature mechanism is used as default when available.
19 To set the feature directory following can be executed once (in Linux):
20 > qmake -set QMAKEFEATURES /usr/include/qtfeatures
21 When possible arnlib.prf and arnlib_config.pri is installed in selected directory.
22 Applications using ArnLib can now at best e.g. use:
23 > ARN += client
24 > CONFIG += arnlib
25 And all needed config for ArnLib is automatically loaded.
26 
27 Local adaptions to ArnLib.pro can be put in the file arnlib_local.pri
28 If needed edit the *.pri / *.pro files to adjust
29 them to your needs. Take care to select your deployment directories.

33 Documentation {#ins_doc}
34 =============
35 
36 The documentation is built by:
37 > qmake <Br>
38 > make doc <Br>
39 
40 ArnLib includes a class documentation, that is available in various formats:
41 
42 * **Html files**
43 * **PDF document** <Br>
44  refman.pdf is built by:
45  > cd doc/latex <Br>
46  > make <Br>
47 * **Qt Compressed Help** (*.qch ) for the Qt assistant or creator. <Br>
48  Load the doc/qthelp/arnlib.qch file into Qt Creator. Start Qt creator and go to
49  Tools > Options, open up Help and Documentation. Click Add and browse for the qch file
50  that was just created, then Apply.
51  It's best to close Qt creator at this point, and restart it.

55 Building ArnLib {#ins_build}
56 ===============
57 
58 The software can be built both by command line and IDE (Qt Creator).
59 When using IDE, don't forget the "make install" step.
60 
61 
62 A) Unix {#ins_buildUnix}
63 -------
64 
65 > qmake <Br>
66 > make <Br>
67 > make install
68 
69 The easiest way of installing this library, is to let it be placed in a standard location
70 for librarys and includes, e.g. /usr/lib and /usr/include/ArnInc.
71 When using a shared library it's path has to be known to
72 the run-time linker of your operating system. On Linux systems read
73 "man ldconfig" (or google for it). Another option is to use
74 the LD_LIBRARY_PATH (on some systems LIBPATH is used instead, on MacOSX
75 it is called DYLD_LIBRARY_PATH) environment variable.
76 
77 If you only want to check the library examples without installing something,
78 you can set the LD_LIBRARY_PATH to the lib directory
79 of your local build.
80 it's also possible to compile the sources together by ArnLibCompile (see Using ArnLib below).
81 
82 The examples is built this way:
83 > cd examples/ArnDemoChat <Br>
84 > qmake <Br>
85 > make

161 Using ArnLib {#ins_usage}
162 ============
163 In ArnLib the arnlib.prf -file contains template lines that can be used in the *.pro file of the application.
164 
165 This will give a starting point for the configuration. It works well when using the same
166 base directory for ArnLib as the application, e.g. basedir/ArnLib and basedir/myApp. In
167 Unix-alike systems it's also needed to install the library files in a path known by the system,
168 see a) Unix.
169 
170 When Qt Features is used (default), the applications using ArnLib can e.g. use:
171 > ARN += client
172 > CONFIG += arnlib
173 And all needed config for ArnLib is automatically loaded.
174 
175 It's possible to include the ArnLib source in the application compiling by adding
176 ArnLibCompile to CONFIG. The included part of the source can be selected by addings to ARN,
177 e.g. ARN += server.
178 
179 WARNING! Using source inclusion (static linking) excludes the right to use LGPL for ArnLib.
180 Options are then to use GPL for the whole application or have a written agreement with
181 Michael Wiklund for other terms using the ArnLib.
182 
183 
184 Internal mDNS (ZeroConfig) is selected by adding mDnsIntern to CONFIG.
185 
186 If you don't use qmake you have to add the include path to find the ArnLib
187 headers to your compiler flags and the ArnLib library to your linker list.
